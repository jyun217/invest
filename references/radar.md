# 趋势雷达模块

## 核心目标

**发现"正在发生但尚未被充分定价"的投资方向，并按优先级排序输出可执行的行动方案。**

---

## 数据获取流程

```
Step 1: 政策面扫描（WebSearch）
  → "国务院常务会议 [最近月份] 2025"
  → "发改委 重大政策 2025 [最近月份]"
  → "[领域] 征求意见稿 2025"

Step 2: 海外映射扫描（WebSearch + 脚本）
  → "NVIDIA earnings data center revenue 2025"
  → "Microsoft Google Meta capex AI 2025"
  → python3 scripts/run.py us NVDA quote
  → python3 scripts/run.py us MSFT quote

Step 3: 技术拐点扫描（WebSearch）
  → "[领域] 技术突破 量产 2025"
  → "[领域] 新标准 发布 升级"

Step 4: 资金动向扫描（脚本 + WebSearch）
  → python3 scripts/run.py a <板块ETF> technical
  → python3 scripts/run.py a <板块ETF> fund_flow
  → "北向资金 加仓 板块 2025"
  → "[板块] 主力资金 净流入"

Step 5: 行业轮动判断（脚本批量）
  → 对多个板块 ETF 分别获取 momentum_20d / momentum_60d
  → 排序：谁刚启动（涨幅小+资金流入）、谁过热（涨幅大+资金流出）

Step 6: 负面信号过滤（WebSearch）
  → "[行业] 产能过剩 风险"
  → "[行业] 政策 收紧 限制"
  → "[板块] 估值 泡沫 过热"

Step 7: 交叉验证 + 排序
  → 多维度信号叠加 = 高确定性
  → 按"确定性 × 预期收益 / 定价程度"排序
```

### 维度二：海外映射

**核心逻辑**：海外巨头投钱 → 中国供应链受益 → A股/港股映射标的上涨

**传导链条分析**（不能只看"谁受益"，还要分析传导的确定性和时间差）：

| 传导类型 | 确定性 | 时间差 | 示例 |
|----------|--------|--------|------|
| 直接供应商 | ★★★★★ | 0-3个月 | NVIDIA GPU → 光模块厂直接供货 |
| 间接上游 | ★★★★ | 3-6个月 | 数据中心扩建 → PCB/连接器 |
| 生态受益 | ★★★ | 6-12个月 | AI应用爆发 → 国内软件公司 |
| 概念映射 | ★★ | 不确定 | 美股涨 → A股同概念炒作 |
| 反向受损 | ❌ | 即时 | 制裁/限制 → 受损环节 |

**重点跟踪**：

| 海外公司 | 脚本获取 | 关注指标 | A股/港股映射方向 |
|----------|----------|----------|-----------------|
| NVIDIA | `run.py us NVDA quote` | 股价趋势、市值 | 光模块、PCB、液冷 |
| Microsoft | `run.py us MSFT quote` | 股价、云业务增速 | 数据中心全链 |
| Google | `run.py us GOOGL quote` | Cloud capex | 光模块、交换机 |
| TSMC | `run.py us TSM quote` | 营收趋势 | 半导体设备、封装 |
| Tesla | `run.py us TSLA quote` | 交付量、储能 | 电池、电驱 |
| Apple | `run.py us AAPL quote` | 新品周期 | 果链公司 |

**搜索模板**：
```
"NVIDIA earnings data center revenue 2025"
"Microsoft Azure capital expenditure 2025"
"big tech capex AI infrastructure 2025"
"[公司] earnings call transcript Q[X] 2025"
"[公司] guidance outlook 2025"
```

**港股 vs A股映射差异**：
- 港股反应快（与美股同步），A股滞后1-3个月
- 港股估值通常更低，A股有溢价

```

"融资余额 增加 行业"
```

### 维度五：行业轮动（新增）

**核心逻辑**：市场资金有限，板块之间存在轮动规律。找到"刚启动"的板块，回避"过热"的板块。

**板块状态分类**（基于脚本 technical 数据）：

| 状态 | momentum_20d | momentum_60d | 资金流向 | 操作 |
|------|-------------|-------------|----------|------|
| **刚启动** | +3~+10% | <+10% | 净流入 | ★ 重点关注 |
| 加速中 | +10~+20% | +10~+30% | 净流入 | 可参与但不追高 |
| 过热 | >+15% | >+30% | 开始流出 | ⚠️ 回避，等回调 |
| 调整中 | -5~-15% | 仍为正 | 净流出 | 等止跌再观察 |
| 底部区域 | <-5% | <-10% | 开始流入 | 👀 关注反转信号 |

## 信号综合评估与排序

### 单信号评分

| 维度 | ⭐⭐⭐ 强(5分) | ⭐⭐ 中(3分) | ⭐ 弱(1分) |
|------|---------------|-------------|------------|
| 确定性 | 多维度交叉验证 | 逻辑清晰但缺验证 | 早期概念 |
| 定价程度 | 尚未定价(涨幅<10%) | 部分定价(10~30%) | 已充分定价(>30%) |
| 时间窗口 | 3-6个月兑现 | 6-12个月 | >12个月 |
| 资金支持 | 资金持续流入 | 资金中性 | 资金流出 |
| 轮动位置 | 刚启动/底部反转 | 加速中 | 过热 |

### 优先级排序公式

```
优先级 = 确定性 × 时间窗口得分 × (1 - 定价程度) × 资金方向加成
```

简化判断：
- **立即行动**：确定性⭐⭐⭐ + 未定价 + 资金流入 + 刚启动
- **密切跟踪**：确定性⭐⭐ + 部分定价 + 资金中性
- **暂时观望**：确定性⭐ 或 已充分定价 或 资金流出
- **排除**：命中负面信号

## Actionable 下一步（新增）

**每个信号必须给出明确的下一步操作**：

| 信号强度 | 下一步 |
|----------|--------|
| ⭐⭐⭐ + 未定价 + 资金流入 | → 立即拆解产业链（industry 模块）→ 找到具体标的 → 技术面确认入场时机 |
| ⭐⭐⭐ + 部分定价 | → 拆解产业链 → 重点找定价不充分的细分环节 |
| ⭐⭐ + 资金流入 | → 加入观察清单 → 设置提醒：等催化事件确认后再深入 |
| ⭐⭐ + 资金中性 | → 持续跟踪 → 等待资金信号转正 |
| ⭐ 或负面信号 | → 不投入时间，等待更明确的信号 |
