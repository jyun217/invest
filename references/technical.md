# 技术面分析模块

**技术面是战术工具——回答"怎么打赢"这个问题。多数时候，我们赚的都是趋势的钱。**

## 数据获取

技术面分析通过脚本获取结构化数据，不再依赖网页爬取：

```bash
# A股
python3 scripts/run.py a <代码> technical
python3 scripts/run.py a <代码> fund_flow

# 港股
python3 scripts/run.py hk <代码> technical
python3 scripts/run.py hk <代码> fund_flow

# 美股
python3 scripts/run.py us <ticker> technical
```

脚本返回 JSON，包含均线、MACD、RSI、KDJ、布林带、ATR、成交量等全部指标。

## 多因子技术面评分框架

### 评分权重

```
技术面总分 = 趋势方向(30%) + 资金面(30%) + 市场情绪(20%) + 周期位置(10%) + 板块比较(10%)
```

### 维度一：趋势方向（30%）

**数据来源**：脚本 `technical` 模块返回的 `ma_arrangement`、`deviation_60`、`macd`

| 分数 | 条件 |
|------|------|
| 5 | 均线多头(bullish) + 偏离度 +5~+15% + MACD DIF>0 且 signal=golden_cross |
| 4 | 均线多头(bullish) + 偏离度 +5~+20% + MACD DIF>0 |
| 3 | 均线缠绕(tangled) + 偏离度 -5~+5% + MACD 零轴附近 |
| 2 | 均线空头(bearish) + 偏离度 -5~-20% + MACD DIF<0 |
| 1 | 均线空头(bearish) + 偏离度 < -20% + MACD DIF<0 且 signal=death_cross |

**判断规则**：
- `ma_arrangement` = "bullish" → 趋势向上基础分 3+
- `deviation_60` 在 +5~+15% → 健康偏离
- `deviation_60` > +20% → 过热警告
- `macd.signal` = "golden_cross" 且 `dif` > 0 → 加分

### 维度二：资金面（30%）

**数据来源**：脚本 `fund_flow` 模块

| 分数 | 条件 |
|------|------|
| 5 | 主力5日净流入>0 + 北向连续买入 + 融资余额增加 |
| 4 | 三项中两项为正 |
| 3 | 资金面中性，无明显方向 |
| 2 | 三项中两项为负 |
| 1 | 主力持续流出 + 北向卖出 + 融资余额减少 |

**判断规则**：
- `main_net_inflow_5d` > 0 → 主力买入 (+1)
- `north_net_flow_5d` > 0 → 北向买入 (+1)
- `margin_balance_change_5d` > 0 → 融资增加 (+1)
- 基础分 2，每满足一项 +1，最高 5

**港股特别规则**：
- 港股无北向资金和融资数据，资金面仅参考南向资金
- 数据不完整时，该维度取中性值 3 分

**美股特别规则**：
- 美股无资金流向数据，该维度权重转移到其他维度
- 权重调整为：趋势方向(40%) + 市场情绪(30%) + 周期位置(15%) + 板块比较(15%)

### 维度三：市场情绪（20%）

**数据来源**：脚本 `technical` 模块中的 `volume_state`、`volume_ratio_5_20`、`atr`

| 分数 | 条件 |
|------|------|
| 5 | 换手率适中(3-8%) + ATR收敛(atr_pct<3%) + 温和放量(ratio 1.2~1.5) |
| 4 | 成交量正常(ratio 0.9~1.5) + 量价配合 |
| 3 | 缩量横盘(volume_state="light") |
| 2 | 过度放量(ratio>2.0) 或 极度缩量(ratio<0.5) |
| 1 | 恐慌性放量(ratio>3.0) + ATR极高(atr_pct>6%) |

**判断规则**：
- `volume_state` = "normal" 且 ratio 1.0~1.5 → 量价健康 (4分)
- `volume_state` = "heavy" 且趋势向上 → 资金入场 (5分)
- `volume_state` = "heavy" 且趋势向下 → 恐慌出逃 (1分)
- `atr_pct` < 2% → 波动小，情绪稳定 (+0.5)
- `atr_pct` > 5% → 波动大，情绪极端 (-0.5)

### 维度四：周期位置（10%）

**数据来源**：脚本 `technical` 模块中的 `rsi`、`kdj`、`boll`

| 分数 | 条件 |
|------|------|
| 5 | RSI 40-60 + KDJ 金叉(k>d且j向上) + 布林带中轨附近(position_pct 40-60%) |
| 4 | RSI 30-70 + 非超买超卖区 |
| 3 | 中性区间 |
| 2 | RSI>70 或 <30 + 接近超买/超卖 |
| 1 | RSI>80 或 <20 + KDJ极端 + 布林带边缘(>90%或<10%) |

**判断规则**：
- `rsi.rsi12` 在 40-60 → 中性偏强 (基础分 4)
- `rsi.rsi12` > 70 → 超买警告 (最高 2 分)
- `rsi.rsi12` < 30 → 超卖机会 (关注反转，但不急)
- `kdj.j` > `kdj.k` > `kdj.d` 且 k < 80 → 金叉向上 (+1)
- `boll.position_pct` 40-60% → 中轨区间，方向待确认

### 维度五：板块比较（10%）

**数据来源**：需要额外搜索行业指数数据，或比较个股 `momentum_20d` 与行业 ETF

| 分数 | 条件 |
|------|------|
| 5 | 个股20日涨幅 > 行业指数 + 10% 以上（强alpha） |
| 4 | 个股跑赢行业 5-10% |
| 3 | 个股与行业基本同步（±5%） |
| 2 | 个股跑输行业 5-10% |
| 1 | 个股大幅跑输行业 > 10% |

**获取方式**：
```bash
# 获取行业 ETF 的动量数据做对比
python3 scripts/run.py a <行业ETF代码> technical
# 比较 momentum_20d 差值
```

若无法获取行业数据，该维度取中性值 3。

## 综合评分与趋势阶段映射

| 总分 | 趋势阶段 | 操作建议 |
|------|----------|----------|
| 4.0-5.0 | 强势/健康上行 | 买入/持有 |
| 3.0-3.9 | 偏多/震荡偏强 | 轻仓试探/持有 |
| 2.0-2.9 | 震荡/偏弱 | 观望 |
| 1.0-1.9 | 弱势/超跌 | 回避/等待 |

---

## 趋势五阶段模型（保留）

| 阶段 | 定义 | 核心特征 | 操作 |
|------|------|----------|------|
| **强势上行** | 趋势确认且加速 | 均线多头 + 偏离度>+15% + 放量 | ⚠️ 警惕过热，不追高 |
| **健康上行** | 稳健上升趋势 | 均线多头 + 偏离度+5~+15% + 量价配合 | ✅ **最佳持有/买入** |
| **震荡整理** | 方向不明 | 均线缠绕 + 偏离度±5% + 缩量 | ⏸️ 观望，等方向 |
| **弱势下行** | 下跌趋势确认 | 均线空头 + 偏离度-5~-20% + 反弹无力 | ❌ 不做多，止损 |
| **超跌筑底** | 跌无可跌 | 偏离度<-20% + 极度缩量 + 利空不跌 | 👀 关注转向，不急 |

**阶段判断基于脚本数据的映射**：
- `ma_arrangement` = "bullish" + `deviation_60` > 15% → 强势上行
- `ma_arrangement` = "bullish" + `deviation_60` 5~15% → 健康上行
- `ma_arrangement` = "tangled" + `deviation_60` ±5% → 震荡整理
- `ma_arrangement` = "bearish" + `deviation_60` -5~-20% → 弱势下行
- `ma_arrangement` = "bearish" + `deviation_60` < -20% → 超跌筑底

## 筹码峰与公允价格

**筹码峰 = 公允价格 = 历史上多空双方认可的价格区域**

### 识别方法（基于脚本数据）
- **MA60 附近**：`ma.ma60` 值 = 中期筹码密集区的近似
- **布林带中轨**：`boll.mid` = 20日筹码中枢
- **前期高低点**：`high_52w` / `low_52w` 附近的整数关口

### 支撑/压力判断
- **下方支撑**：MA60、布林带下轨、52周低点
- **上方压力**：前高、布林带上轨、52周高点
- **回踩 MA20/MA60 = 买入机会**（安全边际高，止损明确）

---

## 买入三条件

```
1. ✓ 商业模式靠谱（基本面OK）
2. ✓ 趋势处于"健康上行"阶段（技术面总分 ≥ 3.5）
3. ✓ 股价回撤至重要支撑位附近（MA20/MA60/布林带中下轨）

前提：基本面与趋势方向一致（都向上）
```

### 最佳买入时机的量化条件

| 条件 | 量化标准（基于脚本数据） |
|------|--------------------------|
| 趋势确认 | `ma_arrangement` = "bullish" + `macd.dif` > 0 |
| 回调幅度合适 | 从近期高点回撤，当前价格接近 `ma.ma20` 或 `boll.mid` |
| 缩量回调 | `volume_state` = "light" 或 "normal" |
| 未超卖 | `rsi.rsi12` > 35 |
| 资金未撤 | `main_net_inflow_5d` > 0 或 中性 |

## 风险管理

### 核心规则
1. **趋势优先**：当技术面总分 < 2.5 时，无论基本面多好，不做多
2. **计划先行**：买入前明确止损位、目标位、仓位
3. **依计划行事**：不因情绪违反计划

### 止损位设定（基于脚本数据）
- **个股**：跌破 `ma.ma60` 或 `boll.lower` = 离场信号
- **固定比例**：-7% ~ -10%
- **均线转向**：`ma_arrangement` 从 "bullish" 变为 "tangled"/"bearish" = 减仓

### 盈利保护
- 盈利 > 15%：止损上移至成本价
- 盈利 > 30%：止损上移至 +15%
- 分批止盈：1/3 → 1/3 → 剩余让利润奔跑
