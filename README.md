## 功能概览

| 你说 | Skill 做什么 |
|------|-------------|
| "有什么方向" | 三维趋势雷达扫描（政策/海外映射/技术拐点） |
| "拆解XX产业链" | 上游→中游→下游全链分析，找瓶颈/增量环节 |
| "分析XX公司" | 商业模式 + 财务表现 + 市场定价 + 技术面 |
| "出研报" | 产业定位→基本面→技术面→综合打分→操作建议 |

### 核心分析框架

```
大趋势确认 → 产业链拆解 → 瓶颈/增量环节 → 验证供需 → 选股定性 → 择时定量
```

- **基本面**（定性）：回答"值不值得参与"
- **技术面**（定量）：回答"怎么交易"
- 两者方向一致时才行动

## 五大分析模块

### 1. 趋势雷达

扫描三个维度发现投资机会：

- **国家政策**：国务院/发改委/工信部最新政策动态
- **海外映射**：NVIDIA/Microsoft/Google 等巨头 capex 方向 → A股/港股映射
- **技术拐点**：成本曲线突破、新产品发布、行业标准升级

输出：每个信号标注确定性、定价程度、时间窗口。

### 2. 产业链分析

给定一个行业/趋势，自动：
1. 拆解为上游→中游→下游→配套
2. 评估每个环节的壁垒、格局、受益弹性
3. 找到瓶颈型或增量型环节
4. 输出受益标的清单

### 3. 基本面分析

三维度评估：
- **商业模式**：护城河类型、"伟大公司四条件"检验
- **财务表现**：营收增速、EBIT Margin、现金流
- **市场定价**：P/E、P/S 历史分位 + 同行对比

### 4. 技术面分析

趋势五阶段模型：

| 阶段 | 操作建议 |
|------|---------|
| 强势上行 | 不追高 |
| **健康上行** | **最佳买入/持有** |
| 震荡整理 | 观望 |
| 弱势下行 | 不做多 |
| 超跌筑底 | 关注转向，不急 |

### 5. 综合研报

按标准模板输出完整研报：

```
产业定位 → 基本面 → 技术面 → 综合评分 → 操作建议 → 风险提示
```

评分体系：产业景气度(30%) + 公司质地(40%) + 技术位置(30%) = 总分/5

## 设计原则

| 原则 | 说明 |
|------|------|
| 定性为主 | 商业模式/产业链逻辑推演为核心，量化数据为辅助验证 |
| 数据诚实 | 有数据用数据，没数据不编造，缺失数据标注 N/A |
| 基本面+技术面一致 | 两者方向一致时才给出操作建议 |
| 风险优先 | 每份分析必须包含风险提示和数据局限性说明 |
| 图形优先 | 当趋势指标与主观预期矛盾时，尊重趋势方向 |

## 自定义与扩展

你可以通过修改 `references/` 目录下的文件来定制分析框架：

- 调整评分权重 → 编辑 `report-template.md`
- 添加新的数据源 → 编辑 `data-sources.md`
- 修改风险管理规则 → 编辑 `risk-management.md`
- 添加行业分析模板 → 在 `examples/` 添加新文件

## 常见问题

### Q: 支持哪些市场？
A: 主要聚焦 **A股** 和 **港股**，也可以分析美股但数据获取能力有限。

### Q: 数据来源是什么？
A: 通过 WebSearch 搜索公开财经网站（东方财富、同花顺、雪球、新浪财经等），所有数据标注来源和时间。

### Q: 能替代专业投顾吗？
A: 不能。本 Skill 定位为**投资研究辅助工具**，所有输出仅供参考，不构成投资建议。投资有风险，入市需谨慎。

### Q: 如何提高分析质量？
A: 尽量给出明确的分析对象（公司名称+代码），以及你关心的具体维度（基本面/技术面/产业链等）。
